Документ "Заказ покупателя"

Документ предназначен для оформления намерений покупателя приобрести товары, работы, услуги.

В зависимости от установленного вида операции с помощью данного документа может быть оформлен заказ на продажу или заказ на переработку.

 

Необходимо заполнить следующие реквизиты:

  • Организация. В данном реквизите указывается собственная организация, от имени которой оформляется документ. Выбирается из справочника "Организации". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по нескольким организациям.
  • Вид операции. Указывается вид операции документа.
  • Контрагент. В данном реквизите указывается информация о контрагенте-заказчике. Выбирается из справочника "Контрагенты".

  • Договор. В данном реквизите указывается договор контрагента, в рамках которого оформляется документ. Выбирается из справочника "Договоры контрагентов".

  • Проект. В данном реквизите указывается проект, в рамках которого оформляется заказ.

  • Склад (резерв). В данном реквизите указывается склад, на котором будет зарезервированы запасы при проведении заказа покупателя.

 

На форме документа расположена гиперссылка "Редактировать цены и валюту", по которой вызывается диалог редактирования видов цен, скидок и валюты данного документа.

В подвале формы можно указать ручную скидку на документ, процентом или суммой.


В командной панели табличной части "Запасы" расположена кнопка "Остатки и резервы" (в виде глаза). Данная кнопка активирует расширенный режим позволяющий видеть остатки, резервы и обеспечение по заказу.

При активации режима может наблюдаться замедление работы части функций в документе.


  • Запасы. Данная табличная часть может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части), сканированием штрихкода либо методом ручного ввода. Реквизиты табличной части:

    • Номенклатура - позиция номенклатуры, на которую оформляется заказ.

    • Характеристика - при необходимости в данном поле указывается характеристика номенклатуры. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Количество - количество номенклатурной позиции.

    • В резерв - количество резервируемых запасов на складе указанном в поле "Склад (резерв)

    • Ед. - единица измерения номенклатурной позиции. Выбирается из справочника "Классификатор единиц измерения" или "Единицы измерения".

    • Отгружено - отгруженная номенклатура в рамках заказа.

    • Не отгружено - осталось отгрузить номенклатуры в рамках заказа.

    • Зарезервировано (всего) - зарезервировано номенклатуры в рамках заказа на всех складах.

    • Размещено - размещено (заказано) номенклатуры в документах "Заказ на производство" и "Заказ поставщику" по данному заказу.

    • Не обеспечено - не зарезервировано и не заказано номенклатуры по заказу.

    • Свободно (всего) - количество свободной номенклатуры на всех складах.

    • Склад (резерв) - Склад на котором будет произведен резерв при вводе значения в колонку "В резерв". Отображается при переключении расположения склада в документах отгрузки с шапки на табличную часть.

    • Свободно для резерва - свободное количество номенклатуры на складе из колонки "Склад (резерв)".

    • К поступлению - ожидаемое количество поступления свободной (не привязанной к заказам) номенклатуры.

    • Цена - цена отгрузки номенклатурной позиции.

    • % Скидки - процент ручной скидки (наценки).

    • % Авт. - процент автоматической скидки (наценки). Процент автоматической скидки редактируется в справочнике "Виды автоматических скидок и наценок". Расчёт автоматических скидок в документе осуществляется при записи документа и при нажатии кнопки "% Авт." над табличной частью.

      Сумма - стоимость заказываемой номенклатуры по данной строке.

    • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

    • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

    • Всего - поле, автоматически заполняемое суммарной стоимостью запасов согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

    • Спецификация. Определяет вариант изготовления заказываемой продукции.
    • Содержание - представление запаса в печатной форме. Чтобы реквизит стал видимым, необходимо выполнить "Все действия" - "Изменить форму" и установить флаг напротив реквизита.

 

  • Работы. Табличная часть доступна только для вида операции "Заказ-наряд". Данная табличная часть может быть заполнена с помощью подбора (кнопка "Подбор" командной панели табличной части), сканированием штрихкода либо методом ручного ввода. Реквизиты табличной части:

    • Вид работ - определяет стоимость нормо-часа работы.
    • Работа, услуга - выполняемая работа, оказываемая услуга. 

    • Характеристика - при необходимости в данном поле указывается характеристика работы, услуги. Выбирается из справочника "Характеристики номенклатуры". Реквизит отображается на форме, если в настройках параметров учета включено ведение учета по характеристикам номенклатуры.

    • Норма времени - норма выполнения работы. Для работ по фиксированной стоимости норма времени равна единице.

    • Кратность - количество работ, норма которого указана.

    • Коэффициент - повышающий коэффициент на особенность работ.

    • Цена - цена работы, услуги.

    • % Скидки - процент ручной скидки (наценки).

    • % Авт. - процент автоматической скидки (наценки). Процент автоматической скидки редактируется в справочнике "Виды автоматических скидок и наценок". Расчёт автоматических скидок в документе осуществляется при записи документа и при нажатии кнопки "% Авт." над табличной частью.

      Сумма - стоимость по данной строке.

    • Ставка НДС - ставка налога на добавленную стоимость. Выбирается из справочника "Ставки НДС".

    • Сумма НДС - сумма налога на добавленную стоимость по данной строке.

    • Всего - поле, автоматически заполняемое суммарной стоимостью работ и услуг согласно настройкам цен и валют. Настройки устанавливаются в диалоге, вызываемом по гиперссылке "Редактировать цены и валюту".

    • Спецификация. Определяет вариант выполнения работы. По спецификации можно автоматически заполнить использованные материалы.

    • Материалы - затраченные материалы.

    • Исполнители - сотрудники, выполняющие работы с указанием тарифа.

    • Содержание - представление работы в печатной форме. Чтобы реквизит стал видимым, необходимо выполнить "Все действия" - "Изменить форму" и установить флаг напротив реквизита.

 

  • Доставка. Закладка доступна для вида операции "Заказ покупателя". Реквизиты закладки:

    • Получатель - ФИО получателя заказа

    • Номер телефона, запасной телефон, почта - контактные данные получателя заказа

    • Способ - способ доставки заказа получателю. При невыбранном значении или режиме "Самовывоз" доставка не выполняется.

    • Дата и время - хранит подходящую получателю дату и промежуток времени доставки

    • Служба доставки - используемая курьерская компания (КК). Может быть выбран предопределенный элемент "Доставка собственными силами". Выбирается из справочника "Службы доставки".

    • Курьер - физическое лицо, выполняющее доставку заказа получателю или на склад КК. Выбирается из справочника "Физические лица".

    • Зона / тариф - зона доставки или используемый тариф КК. Выбирается из справочника "Зоны/тарифы доставки".

    • Услуга - номенклатура с типом "Услуга", использующаяся для отражения доставки в учете. Обязательный реквизит при использовании способа доставки отличного от "Самовывоз". Выбирается из справочника "Номенклатура".

    • Трек-номер - номер, присвоенный заказу службой доставки

    • Габариты - габаритные размеры заказа. Общий объем рассчитывается автоматически по объему элементов из табличной части "Запасы". Может быть пересчитан по заданным фактическим габаритам или отредактирован вручную.

    • Вес - вес заказа. Рассчитывается автоматически по весу элементов из табличной части "Запасы". Может быть отредактирован вручную.

    • Объявленная ценность - стоимость заказа для КК

    • Адрес отправления - фактический адрес склада / организации, с которого будет выполнена отгрузка заказа. При ведении учета по складам задается в контактной информации карточки склада, указанного в поле "Склад (резерв)" на закладке "Дополнительно" заказа покупателя. При отключенном учете по складам используется фактический адрес из контактной информации организации.

    • Адрес самовывоза - адрес склада, из которого будет выполнен самовывоз заказа

    • Адрес доставки - адрес, по которому будет выполнена доставка заказа. При доставке в пункт выдачи также содержит адрес получателя, но служит лишь для подбора ближайшего пункта выдачи заказа

    • ПВЗ - пункт выдачи заказа. Выбирается из справочника "Пункты выдачи заказа". Подчинен справочнику "Службы доставки".

    • Комментарий для службы доставки - дополнительная информация, которая будет передана КК

    • Стоимость - расчетная стоимость доставки. Загружается автоматически при использовании Яндекс-доставки, вычисляется по формулам службы доставки, если они заданы или рассчитывается по цене заданной службы доставки. Недоступно при способе учета "Бесплатная доставка".

    • % НДС, сумма НДС - данные НДС доставки.

Далее расположены кнопки для передачи данных заказа в Яндекс-доставку и просмотра текущего состояния заказа в личном кабинете.

  • Цветовая индикация колонок:
  • Можно отгрузить (всего):

    • Зеленый - доступно к отгрузке в полном объеме

    Оранжевый - доступно к отгрузке частично.

    Красный - не доступно  к отгрузке.

  • Обеспечено (всего):

    • Зеленый - обеспеченно полностью.

    Оранжевый - обеспечено частично.

    Красный - не обеспечено.

  • Расшифровки гиперссылок:

  • Зарезервировано (всего) - отображает резервы по всем структурным единицам.

  • Размещено (всего) - отображает размещение номенклатуры в разрезе документов.

  • Свободно (всего) - отображает свободный (не зарезервированный) остаток по всем структурным единицам.

  • К поступлению - отображает ожидаемое к поступлению количество свободной номенклатуры (Заказы поставщикам не привязанные к Заказам покупателя).

  • Дата поступления - ожидаемая дата поступления свободной номенклатуры. Если ожидается поступление по нескольким документам, в расшифровке отражаются соответствующие документы.


  • Документ "Заказ покупателя" может быть введен на основании справочника:

    • "Контрагенты".
