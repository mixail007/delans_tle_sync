Что такое Ricoh ADL?

  

Ricoh ADL – не имеющее аналогов решение для быстрой загрузки бумажных документов в 1С:Предприятие, позволяющее максимально ускорить трудоемкий процесс ввода первичных документов в 1С.

Ключевые особенности сервиса Ricoh ADL

  • Автоматизация обработки любого количества документов, содержащих сложную табличную форму. Достаточно сделать всего несколько кликов, чтобы перенести данные в информационную базу системы 1С.
  • Программа Ricoh ADL отлично обрабатывает сканированные копии документов даже невысокого качества благодаря технологиям компании ABBYY.
  • Типовые первичные документы (ТОРГ-12, Счет-фактура, Акт выполненных услуг, Счет на оплату, УПД) и паспорта Российской Федерации обрабатываются в системе автоматически.
  • Программа Ricoh ADL совместима с типовыми продуктами платформы 1С:Предприятие для бухгалтерского, кадрового и финансового учета (1С:Бухгалтерия, 1С:Управление производственным предприятием, 1С:Комплексная автоматизация, 1С:Управление торговлей).

 

Поддерживаемые источники

  • файлы в форматах .jpg, .jpeg, .png, .tif, .tiff, .png, .pdf
  • МФУ Ricoh

 

Требования к загружаемым документам

  • Бумажные документы должны быть напечатаны на лазерном или струйном принтере.
  • Распознаванию поддаются только машинописные символы. Рукопечатные вставки или приписки недопустимы.
  • Документ должен иметь четко выраженную табличную часть.
  • Для файлов, полученных путем сканирования бумажных носителей: разрешение 300 dpi; размер электронного документа должен соответствовать размеру оригинального документа (формат А4); цветовой режим - оттенки серого или цветной.
  • Размер отправляемого на сервер распознавания файла не более 22,5 Мб.

Примечание: в случае, если размер электронного документа больше формата A4 или разрешение свыше 600 dpi, для одной страницы документа будет использовано более одной страницы лицензии.

 

Запуск программы

Запуск программы Ricoh ADL в 1С:Предприятие

Для открытия окна программы Ricoh ADL, после запуска 1С в режиме 1С:Предприятие, необходимо из каталога установки программы открыть файл ЗагрузкаДокументов.epf через подменю "Файл - Открыть". Как правило, для конкретного пользователя файл достаточно открыть только один раз, в дальнейшем ссылка на файл доступна в списке последних открытых файлов.

Авторизация или лицензирование программы при запуске

При запуске версии программы Ricoh ADL необходимо ввести логин и пароль, полученные при регистрации

Работа c документами

Распознавание данных из файлов или с бумажных носителей осуществляется на сервере распознавания. Дальнейшая обработка документов и загрузка в 1С - в основном окне программы.

Список документов содержит информацию о дате и номере документа, его типе, контрагенте и организации, и сортируется по нескольким параметрам.

Работа с документами в основной форме программы 

На странице отображаются документы, находящиеся на сервере распознавания. 
Список документов выводится постранично с указанным количеством документов на странице (поле "На странице"). Можно указать номер страницы документов для просмотра или использовать кнопки перехода на страницу назад/вперед.
Возможно отображение всех документов в разрезе сервисов "Загрузка документов" и "ДокАрхив".
В разрезе сервиса "Загрузка документов"  документы отображаются по статусам обработки: "Распознанные", "На проверке", "Загруженные", "К перераспознаванию", "Нераспознанные".
В разрезе сервиса "ДокАрхив"  документы отображаются по статусам обработки: "К атрибутированию", "В архиве".
Строка статуса содержит количество документов по статусам. Количество новых документов в разрезе статуса обработки при этом отображается жирным шрифтом.

По нажатию на кнопку "Перераспознать" выделенные флажками  (если не выбраны, то текущий) документы будут отправлены для повторного распознавания. Через время данные перераспознаваемых документов обновятся (время распознавания документа 1-5 минут в зависимости от количества страниц и качества изображений).

По нажатию на кнопку  ("Удалить") выделенные флажками (если не выбраны, то текущий) документы удаляются с сервера распознавания без возможности восстановления.

По нажатию на кнопку "Загрузить в 1С" выделенные флажками (если не выбраны, то текущий) документы будут созданы в учетной системе согласно выбранным настройкам.
Загруженные документы в списке отмечены пиктограммой . При двойном клике на строке загруженного документа открывается документ в учетной системе.

По нажатию на кнопку "Проверить и загрузить" выделенные флажками (если не выбраны, то текущий) и доступные для получения документы сопоставляются с учетной системой для дальнейшей проверки и загрузки.
Документы на проверке в списке отмечены пиктограммами  (при совпадении итоговых сумм) или  (при несовпадении итоговых сумм).
При двойном клике на строке полученного документа он открывается в форме программы для дальнейшей проверки и загрузки.
В полученных с сервера распознавания документах реквизиты автоматически сопоставляются с соответствующими справочниками 1С.
Для получения недоступны:
- документы "в обработке" (уже полученные);
- нераспознанные документы;
- документы неподдерживаемые текущей конфигурацией;
- документы у которых изменен тип документа, порядок или состав страниц (требуют повторного распознавания);
- ранее загруженные документы (для повторной загрузки удалите ссылку на ранее  загруженный документ).

Для выбора сценария работы (обработка входящих или исходящих документов) служит поле "Сценарий работы". Если выбран режим "Автоопределение", то сценарий работы будет определяться автоматически на основании распознанных данных.
Для настройки сценариев и параметров получения и загрузки документов служит форма настроек, вызываемая по кнопке   ("Настройки получения и загрузки документов")

 

Рекомендации для получения удовлетворительного результата распознавания

1. Отправлять на сервер распознавания документы вида:
- ТОРГ-12 (типовая товарная накладная)
- СЧЕТ-ФАКТУРА
- АКТ выполненных услуг
- СЧЕТ на оплату
- УПД
- ПАСПОРТ Российской Федерации.
В результате  распознавания не поддерживаемых видов или документов нестандартной формы документ может быть нераспознан или распознан некорректно.

2. Сканировать бумажные документы, напечатанные на лазерном или струйном принтере без потери качества изображения.
В случае если документ был распечатан с низким разрешением, при заканчивающейся краске, получен  через факсимильный аппарат с потерей качества, документ может быть распознан некорректно.

3. Не вносить в документ рукопечатные вставки или приписки, так как распознаванию поддаются только машинописные символы.
Рукопечатные вставки не распознаются и, зачастую, понижают качество распознавания машинописного текста.

4. Отправлять для распознавания все страницы документа одним пакетом с соблюдением порядка их расположения.
Если нарушен порядок отправки страниц на сервер, то страницы, расположенные перед страницей с шапкой документа не будут распознаны.
При сканировании сторонней программой сохраняйте все страницы в один многостраничный файл или именуйте файлы страниц в алфавитном порядке (например: страница1.jpg, страница2.jpg, страница3.jpg и Т.Д.).

5. Сканировать документы перед соединением степлером или помещением в скоросшиватель.
При сканировании не загибайте часть страницы и не закрывайте часть текста, т.к. при этом могут потеряться идентификаторы документа и его реквизитов, а документ может быть нераспознан или распознан некорректно.

6. Документ должен иметь четко выраженную табличную часть
Не допускается отсутствие рамки или рамка в виде штрихпунктирной линии. В противном случае табличная часть будет нераспознана.
 
7. Для файлов, полученных путем сканирования бумажных носителей:
- оптимальное разрешение 300 dpi (не менее 200 dpi, но не более 600 dpi).
При низком разрешении теряется качество распознавания. Избыточное разрешение увеличивает размер файла изображения, в результате чего снизится скорость отправки и обработки изображения.
В случае если установлено разрешение менее 200 dpi или более 600 dpi будет списано более одной лицензии.
- размер страницы электронного документа должен соответствовать размеру оригинального документа (формат А4).
При  меньшем размере документ может быть нераспознан или распознан некорректно.
В случае если размер больше формата A4, то будет списано более одной лицензии.
- цветовой режим - оттенки серого или цветной
Если документ отсканирован как черно-белый с потерей оттенков серого, качество распознавания будет низким.
Примечание: если документ был отсканирован с низким разрешением, в неправильном цветовом режиме и т.п., после чего был перезаписан с правильными параметрами, то его качество при этом останется низким.

8. Соблюдать правильную ориентацию документа при сканировании
Если изображение перевернуто, то документ может быть нераспознан.

9. Для файлов Excel:
Файлы Excel (*.xls, *.xlsx) должны быть сохранены в версии не ниже Microsoft Excel '97.
Каждый файл Excel должен содержать один документ для распознавания на первом листе книги Excel.
Программа не поддерживает работу с файлами Microsoft Excel '95.
Если файл Excel содержит более одного документа, то будет распознан только первый. Информация находящаяся не на первом листе книги Excel будет проигнорирована.
